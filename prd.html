1. Problem Statement

Plagiarism detection in Ayurvedic research is substantially harder than in conventional biomedical literature. Generic tools like Turnitin produce high false-positive rates because they cannot distinguish domain-specific Sanskrit vocabulary from actual copied content.

1.1 Why Ayurvedic Research Is Different

ChallengeImpact on Generic Tools
Sanskrit & transliterated terms (Dosha, Prakriti, Panchakarma, Rasayana)Identical across all legitimate papers — flagged as matches even in original work
Classical text citations (Charaka Samhita, Sushruta Samhita, Ashtanga Hridayam)Direct shloka quotations are required citation — marked as plagiarism by n-gram tools
Traditional knowledge paraphraseSame canonical concepts described in slightly varied language across hundreds of papers
Multilingual / code-switched contentEnglish + Sanskrit + Devanagari breaks monolingual tokenization pipelines
No centralized Ayurvedic corpusNo PubMed-equivalent; journals are partially indexed, not uniformly searchable

3. Core Features — v1 Scope

3.1 Input Methods

  • Text paste: User pastes article text directly (minimum 200 characters, up to ~5,000 words)
  • URL fetch: User provides a public URL; text is extracted via CORS proxy (allorigins.win)

3.2 AI Semantic Analysis

  • Uses Pollinations AI (free, no account/API key required) with OpenAI-large model via text.pollinations.ai
  • Domain-aware system prompt instructs the AI to ignore Sanskrit/Ayurvedic terminology and classical citations
  • AI distinguishes: legitimate classical citations · domain vocabulary · suspicious paraphrasing · direct copying
  • Returns structured JSON: score, flagged passages, domain terms excluded, classical references, editor guidance

3.3 Plagiarism Score & Report

  • 0–100% score with color-coded ring (green / yellow / orange / red)
  • Score thresholds: 0–15 Minimal · 16–35 Some Similarity · 36–60 Notable · 61–100 High
  • Passage-level flags with confidence level (high / medium / low) and reason
  • Classical references section (green, explicitly not flagged)
  • Domain terms excluded list (prevents false positives)
  • Plain-English editor guidance paragraph

3.4 Domain-Aware False Positive Suppression

  • System prompt explicitly excludes Sanskrit/Ayurvedic terminology from scoring
  • Classical shloka quotations with citation markers labeled "Classical Reference — Not Flagged"
  • Botanical Latin names excluded

5. Technical Architecture

5.1 Stack

LayerTechnologyNotes
FrontendHTML + Tailwind CSS (CDN) + Vanilla JSSingle file, no build step
AI BackendPollinations AI (text.pollinations.ai)Free, CORS-enabled, no auth needed
URL Proxyallorigins.winFree CORS proxy for URL fetching
HostingGitHub PagesFree, auto-deploys from main branch

5.2 Data Flow

  1. User pastes text or fetches URL → text extracted client-side
  2. Frontend sends text to text.pollinations.ai with domain-aware system prompt
  3. AI returns JSON: score + flagged passages + classical refs + excluded terms + editor guidance
  4. Frontend renders color-coded report (no server, no database)
